Comparación de Estructuras HTML: Amazon vs Wikipedia

Análisis del Nivel 3 — HTML Semántico

Similitudes

1. Uso de header, main y footer

Ambos sitios estructuran su página con las tres etiquetas fundamentales del documento HTML5: <header> para el encabezado global, <main> para el contenido principal y <footer> para los datos legales y de navegación final.

2. Navegación múltiple con nav

Ambos utilizan múltiples instancias de <nav>, diferenciadas por el atributo aria-label. Amazon tiene una barra de categorías y menús de footer; Wikipedia tiene navegación del artículo, herramientas, idiomas y pie de página.

3. Barra lateral con aside

Ambos sitios incluyen un <aside>. Amazon lo usa para los filtros de búsqueda; Wikipedia para las herramientas del artículo, la navegación por temas y los enlaces a otros idiomas. En ambos casos el contenido del aside es complementario, no central.

4. Formularios de búsqueda

Ambos incluyen un <form role="search"> en el <header>, con <input type="search"> y <button type="submit">. Amazon añade un <select> de categoría; Wikipedia es más simple.

5. Uso de div como contenedor de diseño

Ambos recurren a <div> para agrupar elementos con propósito de maquetación (filas, columnas, grillas), especialmente en el interior de las secciones semánticas.

6. Uso de span para texto en línea

Ambos usan <span> para marcar fragmentos de texto sin significado semántico propio: contadores, precios, etiquetas decorativas, badges o porciones de texto que necesitan estilo diferenciado.

Diferencias

Comparación punto a punto
Aspecto Amazon Wikipedia
Etiqueta central del contenido Grillas de <article> repetibles (tarjetas de producto). El contenido es una colección, no un documento único. Un único <article> que contiene todo el texto enciclopédico. Es un documento completo y autocontenido.
Uso de section Secciones de página: banner, categorías, recomendaciones. Son bloques de marketing, no partes de un documento. Secciones del artículo: Historia, Marcado, Elementos, Referencias. Respetan la jerarquía del contenido enciclopédico.
Encabezados h1–h6 El <h1> no es visible (o se usa para SEO). Los <h2> y <h3> nombran secciones comerciales y tarjetas de producto. El <h1> es el título del artículo (término enciclopédico). Los <h2>/<h3> estructuran el cuerpo del documento de forma lógica y anidada.
Imágenes Imágenes de producto con alt descriptivo. Logos e iconos decorativos usan aria-hidden="true". Imágenes dentro de <figure> con <figcaption>. Se explica el contexto académico de cada imagen.
aside Panel de filtros de búsqueda (aside externo al artículo). Orientado a la acción del usuario. Doble uso: sidebar de herramientas (fuera del artículo) y ficha informativa incrustada dentro del <article>.
Profundidad del div Alta. Estructura de grillas complejas con múltiples capas de <div> para lograr el layout de e-commerce. Moderada. Los <div> agrupan zonas del layout (sidebar, cuerpo) pero el contenido interno es más semántico.
span Uso frecuente para fragmentos de precio, badges "Prime", contadores de rating, nombres de usuario. Uso para resaltar términos técnicos, números de sección en el índice, fragmentos de código y porciones de texto con idioma diferente (lang).
Accesibilidad Gran cantidad de aria-label para compensar la falta de texto visible en iconos y botones de acción. Uso de aria-label en navegación, pero el contenido es intrínsecamente más accesible por ser texto estructurado con encabezados correctos.

Nivel de complejidad estructural

Amazon — Complejidad: Alta

La página de Amazon es un sistema de interfaz de usuario dinámica orientada a la conversión comercial. Su estructura HTML es profunda, con múltiples niveles de anidamiento de <div> necesarios para grillas de productos, carruseles, filtros y banners. Aunque utiliza etiquetas semánticas (header, nav, main, aside, footer, article), gran parte de la lógica estructural recae en clases de CSS sobre <div> y <span>.

El ratio semántico/div es bajo-moderado: hay semántica en el esqueleto general, pero los componentes internos (tarjetas, carruseles, filtros) dependen fuertemente de contenedores genéricos.

Wikipedia — Complejidad: Media

Wikipedia estructura una página orientada al documento y la lectura. Su HTML es más cercano al ideal semántico: un <article> principal con <section> anidadas, encabezados correctamente jerarquizados, <figure> con <figcaption>, tablas con <th scope>, y referencias con <cite>.

El ratio semántico/div es alto: los <div> y <span> se usan para el layout y la presentación del motor MediaWiki, pero el contenido del artículo es semánticamente rico e independiente del diseño.

Conclusión

Wikipedia es un mejor ejemplo de HTML semántico porque su contenido es fundamentalmente un documento estructurado: los encabezados tienen jerarquía lógica, el artículo es un <article> completo, las secciones respetan el flujo informativo y se usan etiquetas como <cite>, <time>, <figure> y <dfn> para dar significado al contenido.

Amazon prioriza la interfaz funcional y la conversión: su HTML es semántico en el esqueleto (header/main/footer/nav/aside) pero depende de <div> anidados y clases de CSS para construir sus componentes de UI complejos. Esto es comprensible dado su naturaleza de aplicación web dinámica, pero implica un menor aprovechamiento del vocabulario semántico de HTML5.

Ambos sitios demuestran que el HTML semántico y los <div> no son excluyentes: la clave es usar las etiquetas semánticas para el esqueleto y el significado, y reservar <div>/<span> para los contenedores sin significado propio.
